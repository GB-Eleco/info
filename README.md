# 学校信息整合

这是一个可以直接部署到 GitHub Pages 的静态站点，用于展示高校学院报名信息，并支持本地 CRUD。

## 功能

- 左侧按高校档次和省份筛选
- 右侧展示学院条目
- 每个条目都能直接跳转到学院链接
- 报名未开始显示灰色
- 报名已结束显示划线
- 支持新增、编辑、删除，并保存在浏览器本地

## 数据结构

当前站点使用 `data.js` 作为初始数据源，运行后会写入浏览器的 `localStorage`，作为轻量“数据库”。

字段包括：

- 学校名称
- 学院名称
- 所在省份
- 报名开始时间
- 报名截止时间
- 学校档次
- 强/弱 com
- 是否机试
- 学院链接

## 部署到 GitHub Pages

1. 把这些文件推送到仓库根目录。
2. 在 GitHub 仓库的 Settings 里打开 Pages。
3. Source 选 `Deploy from a branch`。
4. Branch 选 `main`，目录选 `/root`。
5. 保存后等待几分钟即可访问。

## 使用说明

- 第一次打开时，示例数据会自动写入本地浏览器。
- 之后在页面里新增、编辑、删除的内容都会保留在当前浏览器。
- 如果清除浏览器数据，本地记录会重置为 `data.js` 里的初始内容。

## 后续扩展

如果你后面想把数据真正接到数据库后端，可以把 `localStorage` 替换成 API 请求，再接入 Supabase、Firebase、Vercel 或自建后端。
